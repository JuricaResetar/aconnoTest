#Introduction

Settings and slots are designed to be a simple interface between uC and a connecting bluetooth device.

Slots are used to setup the advertisement, and settings are used to setup other parameters of embedded system. These parameters can vary in their meaning for the system. There are some general parameters always pressent, like beacon information parameters, but a user of this lib can add his own parameters acording to his needs in the specific project.

#Usage

In order to use slots settings are required, but in order to use settings you dont need slots.

Settings need to be initialized with their init function. After that specific setting or parameter can be accesed with a getter function. These getter functions allow 3 different types of indexing. You can specify either only index of desired setting, or group and group index of desired setting, or name of a setting. This gives flexibility in the way settings are accessed, but note that if accessing settings with names do it as few times as possible. The reason is that all names have to be searched sequentialy which is a time consuming task. If accessing settings with name it is best to save a pointer to that setting localy and then use it through pointer or reference.

Slots are fully configured from the connected device, and are not meant to be altered from within the device on which they run. To use slots only one dedicated thread is needed and they become available over bluetooth connection for configuration. Therefore after run they act as a separate system, so they have no API to interact with them.
